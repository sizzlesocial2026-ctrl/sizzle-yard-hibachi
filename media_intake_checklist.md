# Sizzle Yard Hibachi Media Intake Checklist

Purpose: keep Sizzle Yard Hibachi media honest, separate from Sizzle Social Hibachi, and ready for mobile/desktop website use.

## Brand Gate

- Company: Sizzle Yard Hibachi only.
- Reject any file already used by Sizzle Social Hibachi.
- Unused files from the Sizzle Social folder may be considered only after audit, permission review, and copying into the Yard media folder.
- Reject visible Sizzle Social branding, phone numbers, menus, payment QR codes, uniforms, ad screenshots, or old watermarks.
- Use `sizzle-yard-` file names for approved website assets.

## Permission Gate

- Use only media we own, filmed ourselves, generated for Sizzle Yard, or have clear permission to use.
- Avoid clear customer faces unless permission is confirmed.
- Avoid clear children's faces by default.
- Reject visible addresses, license plates, private documents, or phone numbers.

## Creative Gate

- Sizzle Yard should feel like backyard birthday hibachi, not a luxury chef showcase.
- Preferred balance: guests and party first, food second, chef/fire third.
- Good scenes: laughing guests, birthday moment, small fire moment, games, dancing, table setup, food closeups.
- Avoid scenes where fire looks too large, risky, crowded, or uncontrolled.

## Web Gate

- Desktop hero: 16:9 or 4:3 landscape.
- Mobile hero/poster: 4:5 or 9:16 crop-safe.
- Gallery image: 4:5, 1:1, or 4:3 depending on slot.
- Background video: 5-8 seconds, muted, looped, compressed, with poster image.
- Video cards: 6-15 seconds, muted by default, no fake play UI unless the card actually plays.

## Yearly Refresh Rule

- Do not reuse the same primary website hero image/video for the second year.
- Archive last year's primary campaign visuals before launching the new season.
- Reuse older media only as backup/internal reference, not as main ad or homepage creative.

## Folder Flow

- `media/raw-inbox`: original files received from phone, AirDrop, WeChat, Drive, or camera.
- `media/approved-yard-only`: approved Sizzle Yard originals.
- `media/approved-yard-only/imported-from-social-unused`: approved unused files copied from the Sizzle Social folder for Yard use.
- `media/optimized-web`: cropped/compressed website-ready files.
- `media/rejected-do-not-use`: rejected files kept only for audit.
- `media/source_manifest.csv`: source and usage record for every imported/cross-project media file.

## Before Publishing

- Confirm the page being edited is Sizzle Yard Hibachi.
- Confirm phone, Ads ID, Stripe/deposit behavior, menu, pricing, and logo remain Yard-specific.
- If a file came from the Sizzle Social folder, confirm it is an approved copied file under the Yard project, not a direct reference to the Social project path.
- Test desktop and mobile layout after adding media.
